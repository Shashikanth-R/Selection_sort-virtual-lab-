Selection Sort - Virtual Lab

Definition

Selection Sort is a comparison-based sorting algorithm that repeatedly selects the smallest (or largest) element from the unsorted part and moves it to the beginning.

Theory & Complexity Derivation

The array is divided into two parts: sorted and unsorted. Initially, the sorted part is empty, and the unsorted part is the entire array. The algorithm repeatedly finds the minimum from the unsorted part and moves it to the sorted part.

The total number of comparisons is given by the formula (n(n-1))/2.

Step by Step Process:

      1)Start with the first element. Assume it's the minimum.
      2)Compare it with all other elements to find the actual minimum in the rest of the array.
      3)Once found, swap it with the first element.
      4)Now, the first position is sorted.
      5)Repeat the same for the next position and so on, until the whole array is sorted.

    Time Complexity

    • In the first pass, (n-1) comparisons
    • Second pass: (n-2) comparisons
    • Third pass: (n-3) comparisons
    • ...and so on

    Total comparisons = (n - 1) + (n - 2) + ... + 1 = n(n - 1)/2 = O(nÂ²)

    Best Case: O(nÂ²)
    Average Case: O(nÂ²)

    Space Complexity

    Selection Sort is an in-place algorithm. It does not require additional space for another array.
    Space Complexity: O(1)

    Algorithm

    
\\INPUT: An array of n elements
\\OUTPUT: The array sorted in non-decreasing order 

SelectionSort(arr[], n)
  for i = 0 to n-1
    min_index = i
    for j = i+1 to n
      if arr[j] < arr[min_index]
        min_index = j
    swap arr[i] and arr[min_index]

    Applications

    • Best used when memory writes are costly (minimum swaps)
    • Sorting small lists in embedded systems
    • Educational purposes to teach sorting logic
    • Suitable for linked lists where fewer swaps are beneficial
